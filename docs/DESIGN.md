# 好日子 — 设计文档

## 产品概述

帮助用户挑选良辰吉日的 H5 移动应用，覆盖结婚、开业、搬家、动土、出行、签约等场景，基于中国传统历法（天干地支、五行、黄道吉日）计算，AI 提供通俗解读。

## 技术方案

| 维度 | 选择 | 理由 |
|------|------|------|
| 框架 | React + Vite | 打包快、生态成熟 |
| 历法计算 | 前端 JS 实现 | 万年历是纯数学，可离线 |
| AI 解读 | 火山引擎 CodingPlan (OpenAI 兼容) | 用户已有资源 |
| 部署 | 静态托管 | MVP 阶段无需后端 |

## 设计系统

## 页面结构

### 导航

底部 Tab Bar，3 个入口：
1. **首页** — 场景入口 + 今日黄历
2. **黄历** — 月历详情
3. **我的** — 历史记录（MVP 占位）

### 页面

| 页面 | 功能 |
|------|------|
| 首页 | Logo + 今日黄历大卡片 + 6 场景网格 + 本月吉日日历 |
| 场景详情 | 月份选择 + 生辰输入 + 场景专属选项 |
| 择日结果 | 吉日列表：评分 + 四柱 + 宜忌 + AI 解读 |
| 黄历 | 月历 + 当日详情：宜忌/五行/冲煞/吉神/吉时 |

### 择日场景库

| 场景 | 额外表单字段 |
|------|-------------|
| 结婚 | 新郎新娘生辰(选填)、婚礼类型 |
| 开业 | 法人生辰(选填)、行业类型 |
| 搬家 | 现住址方位 |
| 动土 | 房屋朝向 |
| 出行 | 目地方向 |
| 签约 | 无 |

### 用户流程

```
首页 → 选择场景 → 输入信息 → 查看吉日结果 → AI 解读
首页 → 黄历 Tab → 查看月历 → 切换日期查看详情
```

## 算法架构

### 前端历法计算模块

- 天干地支四柱排盘（年月日时）
- 黄道吉日判定（建除十二神 + 二十八宿）
- 五行纳音
- 冲煞计算
- 吉时推算

### AI 解读模块

- 输入：场景类型 + 四柱信息 + 吉日数据
- Prompt：结合周易知识，用通俗语言解读为什么这天好
- 输出：150-200 字解读文案

## UI/UX 规范

### 图标

- 6 个场景图标：简洁可辨认的线条风格
- 结婚：双环、开业：店铺、搬家：房屋箭头、动土：铲子、出行：指南针、签约：卷轴笔
- Tab 图标：祥云(首页)、日历(黄历)、玉佩(我的)

### 装饰元素

- Logo：篆书「日」字印章 + 太极 S 曲线
- 大卡片背景：回纹网格 + 右下角祥云
- Section 分隔：如意云头 SVG

### 交互规范

- 触控目标 >= 44px
- 卡片 active 缩放 0.96
- 页面切换 fadeIn 0.3s
- 结果卡片 stagger 入场

### 无障碍

- 文字对比度 >= 4.5:1（WCAG AA）
- 宜忌标签：深色背景用白色描边药丸，浅色背景用深色文字
- 焦点态可见
- prefers-reduced-motion 支持
